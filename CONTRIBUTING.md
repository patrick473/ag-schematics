## Getting Started

1. Fork and clone the repository.
2. Install dependencies from the `ag-schematics/` directory:

```bash
cd ag-schematics
npm install
```

## Development Workflow

All commands are run from the `ag-schematics/` directory.

| Command | Description |
|---------|-------------|
| `npm run build` | Compile TypeScript to `dist/` (publishable artifact) |
| `npm test` | Compile to `out/` and run the Jasmine test suite |

The `dist/` folder is what gets published. Never edit it directly.

## Project Structure

```
ag-schematics/
  src/
    collection.json        # Schematic collection manifest — register new schematics here
    versions.ts            # Shared version constants
    <schematic-name>/
      index.ts             # Schematic factory function
      index_spec.ts        # Unit tests
      schema.json          # JSON schema for schematic options
      SPEC.md              # Human-readable spec describing the schematic's behavior
      files/               # Template files generated by the schematic
  utils/                   # Shared helpers (dependency management, JSON editing, etc.)
```

## Adding a New Schematic

1. Create a folder under `src/` named after your schematic.
2. Add the following files:

   - **`index.ts`** — Schematic factory using `@angular-devkit/schematics`.
   - **`schema.json`** — JSON schema describing the schematic's options.
   - **`index_spec.ts`** — Unit tests (see [Writing Tests](#writing-tests)).
   - **`SPEC.md`** — Spec describing behavior, options, generated files, and dependencies.
   - **`files/`** — Any template files the schematic generates.

3. Register the schematic in `src/collection.json`:

```json
"your-schematic-name": {
  "description": "Short description",
  "factory": "./your-schematic-name/index#yourSchematicName",
  "schema": "./your-schematic-name/schema.json"
}
```

4. Add or update any shared version constants in `versions.ts` if your schematic installs npm packages.

## Writing Tests

Tests use **Jasmine** and the `@angular-devkit/schematics/testing` utilities. Place tests in `index_spec.ts` alongside the schematic.

A minimal test looks like:

```typescript
import { SchematicTestRunner } from '@angular-devkit/schematics/testing';
import * as path from 'path';

const collectionPath = path.join(__dirname, '../../collection.json');

describe('your-schematic-name', () => {
  const runner = new SchematicTestRunner('ag-schematics', collectionPath);

  it('generates the expected files', async () => {
    const tree = await runner.runSchematic('your-schematic-name', { /* options */ });
    expect(tree.files).toContain('/expected-file.ts');
  });
});
```

Helper utilities for building test trees are available in `utils/test/tree-helpers.ts`.

Run the full test suite:

```bash
npm test
```

## Submitting a Pull Request

1. Create a branch from `main` with a descriptive name (e.g. `feat/sonar-config-multi-module`, `fix/orval-config-path`).
2. Make your changes — keep each PR focused on a single concern.
3. Ensure all tests pass: `npm test`.
4. Open a pull request against `main` using the provided PR template.
5. Ensure the PR title follows [Conventional Commits](https://www.conventionalcommits.org/) (e.g. `feat:`, `fix:`, `chore:`).

## Reporting Issues

Use the GitHub issue templates:

- **Bug Report** — for problems with an existing schematic.
- **Feature Request** — for improvements to an existing schematic.
- **New Schematic Request** — to propose a brand-new schematic.
